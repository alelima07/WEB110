Forms

Forms are tables that users can fill out to send data back to the server. This contains normal content, markup, special elements called controls and labels after the user is done filling or selecting the form, they complete it by submitting the form to an agent (mail, web server). The form is then usually submitted to the website and the data is used in some way. Your customer details might be stored in a database for later use, or the website might be updated after you choose a product to view. With this you need to set up a webserver to process and receive the data.

An HTML form contains one or more widgets, like text fields, select boxes, buttons, checkboxes, or radio buttons, and all of these are related to a label that explains the purpose and the instruction could be clear.

Before starting your code, it is important to plan your form. It is important to remember that if you donâ€™t plan it ahead or if you make a big form you might have the risk of losing users. So it is important to make a useful form that can collect all the data you need and at the same time fast to fill up.

In order to create a form on HTML, you need the elements: form, label, input, textarea and button. To name the data in a form you need to use the name attribute on each form widget that will collect a specific piece of data.
